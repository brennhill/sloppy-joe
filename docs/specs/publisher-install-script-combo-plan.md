### Phase 1: Data model + extraction

**Files:** `src/registry/mod.rs`, `src/registry/npm.rs`
**Changes:**
- Add `VersionRecord` struct to `registry/mod.rs`
- Add `version_history: Vec<VersionRecord>` field to `PackageMetadata`
- Extract version history from npm `metadata_from_body` (iterate `versions` object, collect publisher + scripts + date for versions within 12 months)
- Other registries default to empty `Vec::new()`

### Phase 2: Signal function + pipeline wiring

**Files:** `src/checks/signals.rs`, `src/checks/names.rs`, `src/checks/metadata.rs`
**Changes:**
- New constant `METADATA_PUBLISHER_SCRIPT_COMBO` in `names.rs`
- New function `check_publisher_script_combo` in `signals.rs`:
  - Walk `version_history` chronologically
  - Find most recent publisher change within 12 months
  - Check if install scripts are present in current version
  - Check if scripts pre-date the publisher change (different message)
  - Return `Option<Issue>` at Error severity
- Wire into `issues_from_lookups` in `metadata.rs`
- Add constant to `all_constants()` test vector

**Estimated size:** ~80 lines

**Automated verification:**
- [ ] `cargo test` — all pass
- [ ] `cargo clippy` — 0 warnings
- [ ] New tests: combo fires when publisher changed 6mo ago + scripts added, does NOT fire when no publisher change, does NOT fire when change >12mo old, does NOT fire when no install scripts, fires with different message when scripts pre-date change, skips when unresolved_version
