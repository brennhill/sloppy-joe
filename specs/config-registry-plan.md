# Plan: Config Registry

> Generated by `/plan` on 2026-03-28. Review before implementation.
> Spec: `specs/config-registry.md`

## Architecture

See `specs/ARCHITECTURE.md` (reviewed 2026-03-28).

**Feature-specific architectural notes:**
- New subsystem: `config::registry` module for registry CRUD, config home resolution, git root detection
- Config pipeline gains a new step: resolve now checks registry + global default before returning None
- `resolve_config_source()` signature changes to accept project directory
- `cache::atomic_write_json()` reused for registry file writes

---

### Phase 1: Module split + registry core
**Files:** `src/config.rs` → `src/config/mod.rs`, new `src/config/registry.rs`
**Changes:**
- Move `config.rs` contents to `config/mod.rs` (mechanical, no behavioral change)
- Update `lib.rs` module declaration from `pub mod config` (file) to `pub mod config` (directory) — Rust handles this automatically
- New `config/registry.rs` with:
  - `config_home() -> PathBuf` — check `XDG_CONFIG_HOME/sloppy-joe/`, then platform default, then `~/.sloppy-joe/` legacy fallback
  - `find_git_root(dir: &Path) -> Option<PathBuf>` — walk up looking for `.git` directory, canonicalize result
  - `load_registry() -> Result<HashMap<String, String>, String>` — read + parse `{config_home}/registry.json`, return empty map if file doesn't exist, error on malformed JSON
  - `save_registry(entries: &HashMap<String, String>) -> Result<(), String>` — atomic write via `cache::atomic_write_json`
  - `register(repo_root: &Path, config_path: &Path) -> Result<(), String>` — canonicalize both paths, validate config path exists, load → insert → save
  - `unregister(repo_root: &Path) -> Result<(), String>` — canonicalize path, load → remove → save
  - `lookup(project_dir: &Path) -> Result<Option<String>, String>` — find git root, check registry, check `{config_home}/default/config.json`
- Tests for all functions: config_home with/without XDG, find_git_root in git repo and non-git dir, register/unregister/lookup round-trip, corrupted registry.json error, missing registry file returns empty map
**Estimated size:** ~200 lines (150 code + 50 test)

### Phase 2: CLI commands + updated resolution
**Files:** `src/main.rs`, `src/config/mod.rs`, `src/lib.rs`
**Changes:**
- Add CLI commands to `main.rs`:
  - `Register { #[arg(long, default_value = ".")] dir: PathBuf, #[arg(long)] config: Option<String> }` — if no config specified, default to `{config_home}/{dirname}/config.json`
  - `Unregister { #[arg(long, default_value = ".")] dir: PathBuf }`
  - `List` — iterate registry, print repo → config path pairs
- Update `Init` in `main.rs`:
  - Add `#[arg(long)] register: bool` and `#[arg(long)] global: bool` flags
  - Without flags: print template to stdout (unchanged behavior)
  - `--register`: create config file at `{config_home}/{dirname}/config.json`, write template, register cwd
  - `--global`: create config file at `{config_home}/default/config.json`, write template
  - `--register --global`: error ("use --register or --global, not both")
- Update `resolve_config_source()` in `config/mod.rs`:
  - New signature: `resolve_config_source(cli_config: Option<&str>, project_dir: Option<&Path>) -> Result<Option<String>, String>`
  - Step 1: `--config` flag
  - Step 2: `SLOPPY_JOE_CONFIG` env var
  - Step 3: `registry::lookup(project_dir)` (only if project_dir is Some)
  - Step 4: return None
- Update `main.rs` Check and Cache command handlers:
  - Call `resolve_config_source(config.as_deref(), Some(&dir))`
  - If None is returned, print error: "No config found for {dir}. Run `sloppy-joe register` to set up per-project config, or pass `--config`." and exit(2)
- Update `lib.rs` callers of `resolve_config_source` if any (currently only main.rs calls it)
- `ensure_config_outside_project` already fires in `load_config_with_project` which is called for all file paths — verify registry-resolved paths flow through this check

### Phase 3: Init --register + end-to-end tests
**Files:** `src/config/registry.rs` (tests), `src/main.rs`
**Changes:**
- Implement `init --register` handler:
  - Find git root, derive dirname
  - Create `{config_home}/{dirname}/config.json` with template content
  - Register cwd → that config path
  - Print confirmation with path
- Implement `init --register --global` error and `init --global` handler:
  - Create `{config_home}/default/config.json` with template content
  - Print confirmation with path
- End-to-end resolution tests (in `config/registry.rs` or `config/mod.rs`):
  - Flag overrides registry
  - Env var overrides registry
  - Registry lookup works
  - Global default fallback works when no registry entry
  - None returned when nothing configured
  - Missing config file at registered path → error
  - Corrupted registry.json → error
  - Not a git repo → error (when no flag/env)
  - Registry-resolved config inside project dir → error (ensure_config_outside_project fires)
- Edge case tests:
  - `register` in non-git directory → error
  - `register --config /nonexistent` → error
  - `unregister` for unregistered repo → no-op or informational message
  - `list` with empty/missing registry → empty output
